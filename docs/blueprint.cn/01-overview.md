# 01 - 项目概述

## 项目简介

RFMarkIt ESP Node 是一款面向动作捕捉和 6DOF 追踪系统的有源标记点固件，基于 **ESP-IDF v5.5.3** 开发。固件运行于 ESP32/ESP32-S3 平台，采集 IMU（惯性测量单元）数据并通过 UDP 实时广播，供上位机系统实现多点位姿估计。

## 硬件变体

| 变体 | MCU | IMU 传感器 | IMU 接口 | 目标芯片 |
|------|-----|-----------|---------|---------|
| **Hardware v1** | ESP32 | HI229 | UART (115200 baud) | `esp32` |
| **Hardware v2** | ESP32-S3 | BNO08X | SPI (3 MHz) | `esp32s3` |

两种变体通过 Kconfig 编译时选择，共享绝大部分上层逻辑代码，仅在 IMU 驱动和引脚定义上有差异。

## 核心功能

1. **IMU 数据采集** — 以 100~400 Hz 频率持续采集加速度、陀螺仪、磁力计和四元数数据
2. **网络数据流** — 通过 UDP 广播（端口 18891）将打包后的传感器数据发送至上位机
3. **设备发现** — 自定义 UDP 发现协议（端口 18889），实现零配置组网
4. **REST API** — HTTP/WebSocket 服务（端口 18888），支持远程配置、调试和 OTA 升级
5. **BLE 配置** — NimBLE GATT 服务，支持通过蓝牙配置 Wi-Fi 凭据
6. **电源管理** — 多级电源状态机，支持深度睡眠和 RTC 唤醒
7. **OTA 升级** — 双分区方案，支持自动回滚
8. **LED 状态指示** — PWM 驱动，支持闪烁、呼吸灯和 Hamming 编码标识

## 版本管理

固件版本号从 Git tag 自动提取（默认 3.3.2），在编译时通过 CMake 注入为预处理器宏，嵌入到固件二进制中。

## 目录结构

```
rfmarkit-esp-node/
├── main/                    # 入口点和 Kconfig 配置
│   ├── app_main.c
│   └── Kconfig.projbuild
├── components/              # 功能组件
│   ├── sys/                 # 系统核心（Wi-Fi、NVS、电源、发现、按钮）
│   ├── apps/                # FreeRTOS 任务实现
│   ├── imu/                 # IMU 硬件抽象层
│   ├── rest_controller/     # HTTP/WebSocket API
│   ├── ble/                 # BLE GATT 服务
│   └── blink/               # LED 控制
├── lib/                     # 底层库
│   ├── bno08x/              # BNO08X SPI 驱动
│   ├── hi229/               # HI229 UART 驱动
│   ├── ring_buf/            # 无锁环形缓冲区
│   ├── libudp/              # UDP 套接字封装
│   ├── libtcp/              # TCP 套接字封装（预留）
│   ├── spatial/             # 空间数学（四元数/旋转矩阵）
│   └── battery/             # 电池电压监测
├── include/                 # 公共头文件
│   ├── modelspec.h          # 硬件引脚定义
│   └── settings.h           # 编译时常量
├── manifests/               # 分区表
│   └── partitions.csv
├── sdkconfig.defaults       # 共享 SDK 配置
├── sdkconfig.defaults.v1    # v1 特定配置
├── sdkconfig.defaults.v2    # v2 特定配置
└── .github/workflows/       # CI/CD
    └── build-firmware.yml
```
